# /inbox — разгрести инбокс

Превращает сырые записи из `99-Inbox.md` в оформленные элементы волта.

## Что делает

1. Читает `99-Inbox.md`
2. Для каждой записи решает:
   - Это **метод**? → создать в `02-Methods/`, обновить таблицу в VibeOS и 00-INDEX
   - Это **факт** для Reference? → добавить в `01-Reference/<раздел>.md` или `04-Memory/facts.md`
   - Это **апдейт карточки** проекта? → обновить `03-Projects/<project>.md`
   - Это **новый проект/направление**? → оставить в 99-Inbox или создать карточку
3. Предлагает план оформления (что, куда, в каком виде)
4. После подтверждения пользователя — создаёт/обновляет заметки
5. Удаляет обработанную запись из 99-Inbox (или помечает как обработанную)
6. Неподтверждённые факты помечает `[проверить]`

## Принципы
- Ничего не выдумывать — только то, что есть в записях
- Спрашивать подтверждение перед действием
- Сохранять wikilink-консистентность
