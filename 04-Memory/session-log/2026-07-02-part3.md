## H. Штатный механизм гейтов: commit-guard плагин

### Контекст
Проблема: /commit гейт можно обойти через прямой `git commit` из bash.
Решение: tool.execute.before в плагинах — неотвратимый гейт на уровне рантайма OpenCode.

### Создан commit-guard.js
- Путь: `serp/.opencode/plugins/commit-guard.js`
- Механизм: tool.execute.before перехватывает bash с "git commit"
- Логика: запускает `python -m pytest -q`, если FAIL → блокирует вызов
- Это НЕОТвРАТИМЫЙ гейт: работает даже при прямом вызове bash из агента

### Git pre-commit hook (дубль)
- Путь: `serp/.git/hooks/pre-commit`
- Логика: pytest перед коммитом (для коммитов вне OpenCode)
- T-051 → done

### Ключевые решения
1. **tool.execute.before** — штатный механизм OpenCode для перехвата инструментов
2. **commit-guard** — блокирует git commit если тесты падают
3. **pre-commit hook** — дубль для коммитов вне OpenCode (прямой git commit)
4. **/commit и verify-в-task** — умные слои сверху (не трогали)
