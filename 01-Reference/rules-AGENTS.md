# AGENTS.md: Файл правил агента

> Проверено: 2026-06-27

## Что это

`AGENTS.md` в корне репозитория — аналог AGENTS.md в волте. Определяет правила работы агента с этим проектом: какие файлы трогать, какие конвенции соблюдать, какие команды доступны.

Не путать с `.opencode/agent/*.md` — там определяется сам агент (модель, права, промпт), а AGENTS.md — это инструкция для агента *о проекте*.

## Структура

```markdown
# Project Name — Agent Instructions

Краткое описание проекта и роли агента.

## Конвенции
- Стиль кода, коммитов, веток
- Запрещённые действия

## Архитектура (кратко)
- Стек, структура директорий
- Команды запуска/тестов

## Окружение
- Переменные среды, секреты
- Зависимости
```

## Пример из проектов волта

| Проект | AGENTS.md | Особенность |
|--------|-----------|-------------|
| dv-hub | Есть | Конвенции TS strict, commit format, submodule flow |
| SERPlux | Есть | Python venv, pytest, bash whitelist |
| vault (этот) | Есть | Правила librarian: только волт, не трогать чужие репо, русский язык |

## Правила
- AGENTS.md должен быть ОДИН в корне проекта.
- Не дублирует .opencode/agent/*.md — те про конфигурацию агента, а AGENTS.md про конвенции проекта.
- Должен читаться агентом при старте сессии.
