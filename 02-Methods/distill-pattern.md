# Distill Pattern

## Проблема
Один и тот же набор действий повторяешь руками каждый раз (прогнать ci → собрать diff → дать ревью). Промптишь одно и то же. Знание «как я это делаю» живёт в голове и теряется.

## Решение
Превращать повторяющийся паттерн в переиспользуемый артефакт. Два уровня (Fowler 2026):
- **Slash-команда** — инструкция, которую запускаешь ТЫ (human-triggered), под частую задачу.
- **Skill** — guidance/instructions/скрипты, которые агент грузит САМ по описанию, когда сочтёт релевантным (LLM-triggered, lazy-load).

Тренд 2026: skills поглощают slash-команды. Но в OpenCode пока оба механизма живут — команды для явного запуска, skill для авто-подгрузки. Аналог `/distill` из MiMo.

## Реализация в OpenCode
Триггер дистилляции: заметил, что промптишь одно в 3-й раз → оформляй.

1. Частая ручная задача → `.opencode/command/X.md` (см. [[commands]]).
2. Контекст, который агент должен подтягивать сам → skill с описанием в SKILL.md. [проверить точную поддержку skills в OpenCode на opencode.ai/docs/skills]

Реальные кандидаты:
- dv-hub: уже дистиллировано — /morning, /spec, /review, /sync-task. Это образец.
- SERPlux: команд НЕТ. Кандидаты: /pipeline-check (прогнать пайплайн на тест-ключах), /review.

## Когда применять / когда НЕ применять
- Применять: паттерн повторился ≥3 раз и стабилен.
- НЕ дистиллировать преждевременно: пока паттерн плавает — оформишь не то. Сначала отладь руками.
- НЕ копировать чужие команды пачками (Fowler: «illusion of control», копипаст из интернета противоречит твоему же контексту).
